    Kapitel 10 – Grundlagen der Programmierung¶

    Was du in diesem Kapitel lernst¶

    • Wie ein einfaches Programm grundlegend aufgebaut ist (Variablen, Datentypen, Operatoren, Kontrollstrukturen)
    • Wie ein „Hello World"-Programm in PHP aussieht und was jede Zeile bedeutet
    • Was die wesentlichen Schritte von der Quelle zum ausführbaren Programm sind
    • Wie Hello World in verschiedenen Sprachen verglichen werden kann (PHP, Python, Java)

    So gehst du vor¶

    1. Lies die Kapitelinhalte und lies den Code aufmerksam – du musst ihn nicht tippen, sondern verstehen.
    2. Bearbeite die Kurzübungen der Reihe nach – von Grundlagen bis Experte.
    3. Arbeite die Workshop-Aufgabe durch. Sie vertieft das Gelernte an einem zusammenhängenden Szenario.

    10.1 Aufbau eines Programms¶

    Ein Programm ist eine geordnete Folge von Anweisungen, die ein Computer ausführen kann. Jede Anweisung wird nacheinander verarbeitet – es sei denn, Kontrollstrukturen verändern den Ablauf.

    Die Grundbausteine fast jedes Programms sind:

    Baustein Beschreibung
    Variablen Benannte Speicherplätze für Werte
    Datentypen Legen fest, welche Art von Wert gespeichert wird
    Operatoren Führen Berechnungen und Vergleiche durch
    Kontrollstrukturen Steuern den Ablauf (Bedingungen, Schleifen)
    Funktionen Wiederverwendbare Codeblöcke
    Kommentare Erläuterungen im Code (werden nicht ausgeführt)

    Alle Codebeispiele in diesem Kapitel sind in PHP geschrieben. PHP ist eine serverseitige Skriptsprache, die im Web weit verbreitet ist und eine gut lesbare, imperative Syntax bietet.

    PHP-Grundregel

    Jede PHP-Datei beginnt mit <?php. Anweisungen enden mit einem Semikolon ;. PHP ist eine Interpretersprache – es wird kein separater Kompilierungsschritt benötigt.

    10.2 Variablen und Datentypen¶

    Variablen in PHP¶

    In PHP beginnen Variablennamen immer mit einem $-Zeichen, gefolgt vom Namen. PHP ist dynamisch typisiert: Der Datentyp wird automatisch aus dem zugewiesenen Wert abgeleitet.

    1
2
3
4
5
6
    <?php
$vorname = "Maria";       // String (Zeichenkette)
$alter = 28;              // Integer (ganze Zahl)
$gehalt = 3250.50;        // Float (Dezimalzahl)
$istAngemeldet = true;    // Boolean (wahr/falsch)
$adresse = null;          // Null (kein Wert)

    Datentypen im Überblick¶

    Typ PHP-Beispiel Beschreibung
    String "Hallo Welt" Text, in Anführungszeichen
    Integer 42 Ganze Zahl
    Float 3.14 Dezimalzahl
    Boolean true / false Wahrheitswert
    Array [1, 2, 3] Liste von Werten
    Null null Kein Wert (Platzhalter)

    Benennung von Variablen¶

    Gute Variablennamen sind: - Aussagekräftig: $kundenname statt $x - Konsistent: Entweder immer camelCase ($kundenName) oder Unterstrich ($kunden_name) - Kein reserviertes Wort: $echo, $if, $while sind nicht erlaubt

    Operatoren verarbeiten Werte und liefern ein Ergebnis.

    Arithmetische Operatoren¶

    Vergleichsoperatoren¶

    Vergleiche liefern immer true oder false.

    Operator Bedeutung Beispiel Ergebnis
    == Gleich (Wert) 5 == "5" true
    === Identisch (Wert + Typ) 5 === "5" false
    != Ungleich 5 != 6 true
    < Kleiner als 3 < 5 true
    > Größer als 3 > 5 false
    <= Kleiner gleich 5 <= 5 true
    >= Größer gleich 6 >= 5 true

    Logische Operatoren¶

    Verknüpfen mehrere Bedingungen.

    Operator Bedeutung Beispiel
    && Und $a > 0 && $b > 0
    \|\| Oder $a > 0 \|\| $b > 0
    ! Nicht !$istAngemeldet

    10.6 Hello World – erstes Programm¶

    „Hello World" ist das klassische erste Programm: Es gibt lediglich den Text „Hello, World!" auf dem Bildschirm aus. Es dient dazu, die grundlegende Funktionsweise einer Programmiersprache und die Entwicklungsumgebung zu testen.

    Hello World in PHP¶

    1
2
    <?php
echo "Hello, World!";

    Was passiert Zeile für Zeile:

    Zeile Bedeutung
    <?php PHP-Startmarker – alles danach ist PHP-Code
    echo Ausgabe-Anweisung – gibt Text auf dem Bildschirm aus
    "Hello, World!" Der auszugebende Text (String)
    ; Beendet die Anweisung (in PHP verpflichtend)

    Hello World im Sprachvergleich¶

    Das Konzept ist dasselbe, die Syntax unterscheidet sich:

    1
2
    <?php
echo "Hello, World!";

    1
    print("Hello, World!")

    1
2
3
4
5
    public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

    1
2
3
4
5
6
    using System;
class Program {
    static void Main() {
        Console.WriteLine("Hello, World!");
    }
}

    Beobachtungen: - PHP und Python: Sehr kurz, direkt ausführbar – typisch für Skriptsprachen - Java und C#: Mehr Rahmencode (Klasse, Main-Methode) – typisch für Compiler-Sprachen mit strikter Struktur

    10.7 Von der Quelle zum ausführbaren Programm¶

    Bei Interpreter-Sprachen (PHP, Python)¶

    flowchart TD
    A["Quellcode schreiben\nz. B. hello.php im Texteditor"] --> B["Interpreter aufrufen\nz. B. php hello.php im Terminal"]
    B --> C["Direkte Ausführung\nZeile für Zeile"]
    C --> D(["Ausgabe sichtbar"])

    Kurzform: Schreiben → Interpreter starten → Ergebnis sehen

    Bei Compiler-Sprachen (Java, C#)¶

    flowchart TD
    A["Quellcode schreiben\nz. B. HelloWorld.java in IDE"] --> B["Kompilieren\nCompiler erzeugt HelloWorld.class / Bytecode"]
    B --> C["Linken\nBibliotheken werden eingebunden"]
    C --> D["Ausführen\nJVM / .NET-Runtime führt das Programm aus"]
    D --> E(["Ausgabe sichtbar"])

    Kurzform: Schreiben → Kompilieren → Ausführen → Ergebnis sehen

    Werkzeuge: Editor vs. IDE¶

    Werkzeug Beschreibung Beispiele
    Texteditor Einfaches Bearbeitungswerkzeug Notepad++, Sublime Text, VS Code
    IDE (Integrated Development Environment) Vollständige Entwicklungsumgebung mit Editor, Debugger, Build-System PhpStorm, Eclipse, IntelliJ IDEA, Visual Studio

    IDEs bieten zusätzliche Funktionen: - Syntax-Highlighting: Schlüsselwörter farblich hervorheben - Auto-Vervollständigung: Vorschläge während der Eingabe - Debugger: Code schrittweise ausführen und Variablen beobachten - Build-System: Kompilieren und Testen auf Knopfdruck

    Kurzübungen¶

    Grundlagen Datentypen erkennen

    Ordne die folgenden Werte dem passenden PHP-Datentyp zu:

    Datentypen: String · Integer · Float · Boolean · Array · Null

    Werte: 1. "Hallo Welt" 2. 42 3. 3.14 4. true 5. null 6. [1, 2, 3, 4] 7. "123" 8. false 9. 0.0 10. ""

    Beantworte außerdem: Was ist der Unterschied zwischen 42 und "42"?

    Tipp

    Anführungszeichen machen aus einer Zahl einen String. Ohne Anführungszeichen ist es eine Zahl.

    Lösung
    1. String – Text in Anführungszeichen
    2. Integer – ganze Zahl
    3. Float – Dezimalzahl
    4. Boolean – Wahrheitswert
    5. Null – kein Wert
    6. Array – Liste von Werten
    7. String – obwohl es wie eine Zahl aussieht, ist es wegen der Anführungszeichen ein String
    8. Boolean – Wahrheitswert (falsch)
    9. Float – Dezimalzahl (auch wenn sie 0 ist)
    10. String – leerer String (nicht null!)

    Unterschied 42 vs "42": 42 ist eine Integer-Zahl, mit der man rechnen kann. "42" ist ein String-Text – man kann damit zwar in PHP bedingt rechnen (automatische Typkonvertierung), aber konzeptionell ist es ein Text, keine Zahl.

    Fortgeschritten PHP-Code lesen und verstehen

    Lies den folgenden PHP-Code und beantworte die Fragen:

     1

    <?php
$preis = 49.90;
$rabatt = 10;
$istMitglied = true;

if ($istMitglied) {
    $endpreis = $preis - ($preis * $rabatt / 100);
} else {
    $endpreis = $preis;
}

echo "Endpreis: " . $endpreis . " Euro";

    Aufgaben:

    1. Was gibt dieses Programm aus? Berechne manuell.

    2. Was würde ausgegeben werden, wenn $istMitglied = false; gesetzt wäre?

    3. Erkläre, was der .-Operator in der letzten Zeile macht.

    4. Was passiert, wenn $rabatt = 0; gesetzt wird? Ändert sich das Ergebnis?

    5. Welche Kontrollstruktur wird in diesem Code verwendet?

    Tipp

    Führe die Berechnung Schritt für Schritt aus: Zuerst $preis * $rabatt / 100, dann $preis minus diesem Wert.

    Lösung

    1. $endpreis = 49.90 - (49.90 * 10 / 100) = 49.90 - 4.99 = 44.91. Ausgabe: „Endpreis: 44.91 Euro"

    2. Der else-Zweig wird ausgeführt: $endpreis = $preis = 49.90. Ausgabe: „Endpreis: 49.9 Euro"

    3. Der .-Operator ist der Verkettungsoperator in PHP. Er verbindet mehrere Strings zu einem einzigen Text-Ausgabewert.

    4. Wenn $rabatt = 0: 49.90 - (49.90 * 0 / 100) = 49.90 - 0 = 49.90. Ausgabe: „Endpreis: 49.9 Euro" – kein Unterschied zu keinem Rabatt.

    5. Selektion (if-else) – abhängig vom Wahrheitswert von $istMitglied wird ein Pfad gewählt.

    Experte Hello World vergleichen und erklären

    Betrachte die folgenden Hello-World-Programme in drei Sprachen:

    PHP: 

    1
2
    <?php
echo "Hello, World!";

    Python: 

    1
    print("Hello, World!")

    Java: 

    1
2
3
4
5
    public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

    Aufgaben:

    1. Erkläre den Unterschied im Umfang des Codes: Warum braucht Java so viel mehr Rahmencode als PHP oder Python?

    2. PHP und Python sind Interpreter-Sprachen, Java ist eine Compiler-Sprache. Erkläre in je 2 Sätzen, wie das „Ausführen" bei beiden Typen abläuft.

    3. Was ist der Zweck eines „Hello World"-Programms? Warum beginnt man in jeder Sprache damit?

    4. Beschreibe in einem Satz: Was ist der wesentliche Schritt „von der Quelle zum ausführbaren Programm" bei einer Interpreter-Sprache?

    5. Was bedeutet der Begriff „IDE"? Nenne zwei Vorteile einer IDE gegenüber einem einfachen Texteditor.

    Tipp

    Java ist eine stark typisierte OOP-Sprache. Jedes Programm muss in einer Klasse stehen, und der Einstiegspunkt heißt immer main.

    Lösung

    1. Java ist eine stark typisierte Objektorientierte Sprache: Jeder Code muss in einer Klasse stehen. Die main-Methode ist der Pflicht-Einstiegspunkt jedes Java-Programms. PHP und Python sind Skriptsprachen, die direkt ohne Klassen-Rahmen ausführbar sind.

    2. Interpreter: Der Interpreter liest den Quellcode Zeile für Zeile und führt ihn sofort aus. Es gibt keinen separaten Build-Schritt. Compiler: Der Compiler übersetzt den gesamten Quellcode vor der ersten Ausführung in Maschinencode oder Bytecode. Erst danach kann das Programm gestartet werden.

    3. „Hello World" testet, ob die Entwicklungsumgebung korrekt eingerichtet ist und die Sprache richtig installiert wurde. Es ist die kleinstmögliche Erfolgskontrolle: Wenn der Text erscheint, funktioniert alles.

    4. Quellcode schreiben → Interpreter aufrufen → direkte Ausführung. (Kein Build-Schritt nötig.)

    5. IDE = Integrated Development Environment (Integrierte Entwicklungsumgebung). Vorteile: Syntax-Highlighting hebt Fehler sofort hervor; Auto-Vervollständigung beschleunigt das Schreiben und reduziert Tippfehler.

    Workshop PHP-Programm analysieren – Notenverwaltung

    Szenario: Du arbeitest als IT-Supporter in einer Berufsschule. Ein Lehrer zeigt dir folgenden PHP-Code für eine einfache Notenausgabe und bittet dich, ihn zu erklären und zu verbessern.

     1

    <?php
// Notenliste für Schüler
$name = "Jan Berger";
$noten = [2, 3, 1, 4, 2, 3];
$summe = 0;
$anzahl = 0;

for ($i = 0; $i < 6; $i++) {
    $summe = $summe + $noten[$i];
    $anzahl = $anzahl + 1;
}

$durchschnitt = $summe / $anzahl;

if ($durchschnitt <= 2.5) {
    $bewertung = "Sehr gut bis gut";
} elseif ($durchschnitt <= 3.5) {
    $bewertung = "Befriedigend";
} else {
    $bewertung = "Verbesserungswürdig";
}

echo $name . ": Durchschnitt " . $durchschnitt . " (" . $bewertung . ")";

    Aufgaben:

    1. Erkläre, was dieses Programm Schritt für Schritt tut (mind. 5 Schritte beschreiben).

    2. Was ist die Ausgabe des Programms? Berechne den Durchschnitt manuell.

    3. Was macht die for-Schleife? Erkläre die drei Teile ($i = 0; $i < 6; $i++).

    4. Identifiziere alle verwendeten Datentypen und Kontrollstrukturen im Code.

    5. Der Lehrer möchte, dass das Programm auch dann funktioniert, wenn in der Zukunft mehr oder weniger Noten eingetragen werden – ohne den Code ändern zu müssen. Welche Zeile müsste angepasst werden und wie?

    6. Welche Informationen würde ein Hello-World-Programm allein noch nicht abdecken, die dieser Code demonstriert?

    Deliverables: Schritt-für-Schritt-Erklärung + Ausgabeberechnung + Antworten auf alle Fragen

    Tipp

    Für Frage 5: PHP hat eine eingebaute Funktion namens count(), die die Anzahl der Elemente in einem Array zurückgibt.

    Lösung

    Bewertungskriterien:

    1. Schritte: (1) Name und Notenliste definieren. (2) Summenvariablen auf 0 setzen. (3) For-Schleife addiert alle Noten. (4) Durchschnitt berechnen. (5) Bewertung per if-elseif-else bestimmen. (6) Ausgabe.

    2. Noten: 2+3+1+4+2+3 = 15. Anzahl: 6. Durchschnitt: 15/6 = 2.5. Bewertung: „Sehr gut bis gut" (2.5 ≤ 2.5). Ausgabe: „Jan Berger: Durchschnitt 2.5 (Sehr gut bis gut)"

    3. $i = 0 = Startwert. $i < 6 = Schleife läuft, solange $i kleiner als 6 ist (also für $i = 0,1,2,3,4,5 – sechs Durchläufe). $i++ = $i nach jedem Durchlauf um 1 erhöhen.

    4. Datentypen: String (\(name), Array (\)noten), Integer (Noten, \(i), Float (\)durchschnitt). Kontrollstrukturen: for-Schleife, if-elseif-else.

    5. $i < 6 und $anzahl berechnung. Besser: for ($i = 0; $i < count($noten); $i++) – dann zählt PHP die Noten automatisch. Auch $anzahl = count($noten) vor der Schleife möglich.

    6. Hello World zeigt nur Ausgabe. Dieser Code zeigt: Variablen verschiedener Typen, Arrays, Schleifen, Berechnungen, bedingte Ausgabe – echte Programmlogik.
